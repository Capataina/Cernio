## Methodologies and Soft Skills

The patterns below are derived from the actual project portfolio, not from a separate LifeOS source — they are the cross-cutting practices observable across multiple projects.

**Plans-driven, externalised-memory engineering.** Every active project carries some form of pre-implementation planning artefact: Cernio's `context/plans/` and 27-finding code-health audit; Image Browser's R-tag annotated 28-finding audit shipped end-to-end with forward + reverse traceability; NeuroDrive's D1–D23 implementation decision log for the M6 build; Nyquestro's 18 KB `IMPLEMENT_NOW_CORE_HARDEN_BOOK_MVP.md` reading as a detailed Phase A + Phase B specification; Vynapse's 19-December-2025 introduction of `plans/NNN_<topic>.md` files; Chrona's formal `plans/` directory with the 11-section per-plan template; Consilium's `IMPLEMENT_NOW_*` execution playbooks acting as working-memory artefacts that outlive active sessions. Plan documents capture rejected alternatives, exit criteria, and per-step checklists rather than just TODO lists.

**Honest-self-assessment / anti-puffing discipline.** Every per-project file in this folder follows the "what the README claims vs what the code does" pattern explicitly. Vynapse's LifeOS Overview explicitly directs against quoting the README's opening line externally. AsteroidsAI's project file enumerates seven concrete cross-method comparability confounds rather than claiming a clean benchmark. Aurix carries an eleven-item gap inventory with named blast radius and fix directions. Chrona's Overview frames the project as "a CMake/CLI/error-model scaffold in C++, planned as a future VCS — not a content-addressed VCS in C++". Tectra's Overview separates README intent from code reality at every step. The discipline is structural, not aspirational.

**Iterative milestone-driven development with named exit criteria.** NeuroDrive's M1–M6 sequence each gated by concrete behaviour (M5: all 8 cars completing the full Monaco loop with crash rate falling 100% → 56%, 96% of crashes anticipated by throttle release > 0.25 s before impact; M6: brain-fleet reward trend rising over ~2000 episodes plus observable directional bias signature plus at least one replacement + one neurogenesis event); Cernio's session 1–9 progression each producing distinct artefacts (sessions 1–7 built core product end-to-end; session 8 added 22-factor location rubric; session 9 added 316 tests + code-health audit + skills migration); Vynapse's 10-milestone roadmap with 16 deliberate 0-byte stub files registered in `mod.rs` to encode roadmap intent visibly; Zyphos's 30-milestone ladder across 7 phases.
