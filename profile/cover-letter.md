Hello,

I am a recent Computer Science graduate based in London with a strong interest in systems engineering, AI infrastructure, and performance-critical software. I enjoy working at the layer where design decisions have real consequences: where the wrong abstraction costs you latency, the wrong data structure breaks under concurrency, or the wrong architecture makes the whole thing impossible to reason about. That is the kind of engineering I find genuinely interesting, and it is what I have been building towards.

During university, I took on the role of lead engineer in an eight-person team developing a simulation game in Java using the LibGDX framework. I was responsible for the overall architecture, set up CI/CD pipelines with GitHub Actions, and implemented automated testing with JUnit across cross-platform test matrices. That experience taught me how to think about systems that need to work reliably for other people, not just on your own machine, and how to communicate clearly in a technical context where everyone has a different mental model of the codebase.

Since graduating, I have been developing a portfolio of self-directed systems projects that each explore a different domain in depth. \textbf{Aurix} is a local-first DeFi analytics platform built in Rust and Tauri, combining concurrent cross-DEX arbitrage detection, Uniswap V3 liquidity pool backtesting with tick mathematics, and quantitative risk modelling, all running on-device with no cloud dependencies. \textbf{NeuroDrive} is a brain-inspired continual learning system where a single persistent agent learns to drive in a custom 2D racing environment using Hebbian plasticity, STDP eligibility traces, and dopamine-modulated weight updates, built entirely from first principles in Rust with no external ML frameworks. \textbf{Image Browser} is a production desktop application for managing large local image libraries, with CLIP-powered semantic and visual similarity search running entirely offline via ONNX Runtime. \textbf{Consilium} is a multi-agent debate orchestration platform built on LangChain that structures discussions between Claude, GPT, and Gemini, compressing dialogue through a rolling summarisation engine and giving all agents shared tool access via MCP. I have also been building \textbf{AsteroidsAI}, a benchmarking platform for comparing NEAT, evolutionary strategies, and deep reinforcement learning on a continuous-control environment, which grew out of a genuine curiosity about how different optimisation philosophies produce different kinds of intelligent behaviour.

These projects were not built to fill a CV. Each one started because I wanted to understand something deeply, and the only way I know how to do that is to build it. Working across systems programming, AI infrastructure, financial engineering, and agent orchestration has given me a broad foundation, but more importantly, it has taught me how to move between domains without losing rigour.

Longer term, I am drawn to the intersection of AI and health. Strength training and nutrition science are central to how I live, and I find the engineering problems in that space genuinely compelling: building systems that can reason meaningfully over personal health data, surface useful insights from physiological signals, and do so in a way that respects privacy and works reliably over time. The kind of work being done in that space is exactly the kind I want to grow into.

I am looking for a role where I can contribute to real infrastructure, work alongside engineers who take correctness and systems design seriously, and keep building in the direction I am already heading.
